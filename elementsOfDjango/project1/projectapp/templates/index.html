Documentation of Django Project Elements

1. Project Directory

This is the root directory of your Django project. It contains the core project files, including settings, URL configuration, and project-specific files.

2. manage.py

Location: Root of the project directory.

Purpose: A command-line utility that helps manage your Django project. It allows you to run server commands like runserver, migrate, startapp, and others.

Usage Example:

  • python manage.py runserver – Starts the development server.
  • python manage.py migrate – Applies database migrations.

3. App Directory

If you create a new Django app, Django generates a folder for it. Each app is responsible for handling specific functionality within the project.

4. __init__.py

Location: Found in both the project and app directories.

Purpose: Indicates that the directory should be treated as a Python package. It's generally an empty file but is essential for Python to recognize the directory as a package.

5. settings.py

Location: Inside the project directory (myproject/myproject/settings.py).

Purpose: This file contains the configuration settings for the Django project, such as database settings, installed apps, middleware, and more.

Key Sections:

  • INSTALLED_APPS: Lists the apps that are installed in your project.
  • DATABASES: Defines the database connection settings.
  • TEMPLATES: Manages settings for the template engine.
  • ALLOWED_HOSTS: Specifies the domains that can serve the app.

6. urls.py

Location: Inside the project directory.

Purpose: This file maps URL paths to views. It defines the URL structure of the entire project.

7. wsgi.py

Location: Inside the project directory (myproject/myproject/wsgi.py).

Purpose: This file provides an entry point for WSGI-compatible web servers (such as Gunicorn or uWSGI) to serve your project in production environments.

8. asgi.py

Location: Inside the project directory (myproject/myproject/asgi.py).

Purpose: Provides an entry point for ASGI-compatible web servers (such as Daphne or Uvicorn) to serve your project. ASGI is used for handling asynchronous protocols such as WebSockets.

9. App-Specific Files in myapp/

  • models.py: Defines the data models for the app. Each model is a Python class that subclasses django.db.models.Model.
  • views.py: Defines the logic for handling requests and returning responses. Each view function is tied to a URL in urls.py.
  • urls.py (App-level): Defines URL routes specific to the app.
  • admin.py: Registers models for the Django admin interface.
  • apps.py: Contains configuration for the app.
  • tests.py: Contains unit tests for the app's functionality.
  • migrations/: Stores migration files that track changes to models over time.

10. db.sqlite3

Location: Root of the project directory.

Purpose: The default SQLite database file that is created when you run migrations for the first time. You can change the database engine in settings.py to use other databases like PostgreSQL or MySQL.

11. __init__.py

The __init__.py file is used to mark a directory as a Python package. Without this file, Python wouldn't recognize the directory as a package, and you wouldn't be able to import files or modules from it in other parts of your project.

12. test.py

The tests.py file in a Django application is where you write unit tests and other types of tests to ensure the functionality of your code.
